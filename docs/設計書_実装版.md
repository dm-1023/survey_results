# アンケート集計レポート 実装設計書

## 1. この設計書の位置づけ

この設計書は、`docs/設計書_2.md` を主設計とし、`docs/設計書_1.md` の詳細な印刷仕様・非機能要件・操作フローを取り込んだ実装用の設計書である。

目的は、紙や手作業で集計済みのアンケート結果をWeb画面で入力し、A4縦向き・白黒印刷に適したレポートとして表示し、ブラウザ印刷からPDF保存または紙印刷できるようにすることである。

このアプリはオンライン回答収集サービスではない。回答者1人ずつの入力、ログイン、共有、サーバー保存、AI要約、OCRは初期リリースの対象外とする。

## 5. データ設計

### 5.1 型定義

`src/types/survey.ts`

```ts
export type QuestionType = "single_choice" | "multiple_choice" | "free_text";

export type Survey = {
  id: string;
  title: string;
  issuer?: string;
  periodText?: string;
  distributedCount?: number;
  collectedCount?: number;
  note?: string;
  questions: Question[];
  createdAt: string;
  updatedAt: string;
};

export type Question = {
  id: string;
  questionText: string;
  questionType: QuestionType;
  options: QuestionOption[];
  freeTextAnswers: FreeTextAnswer[];
  freeTextSummary?: string;
  sortOrder: number;
};

export type QuestionOption = {
  id: string;
  optionText: string;
  count: number;
  sortOrder: number;
};

export type FreeTextAnswer = {
  id: string;
  text: string;
  sortOrder: number;
};
```

### 5.2 型設計のルール

* `options` と `freeTextAnswers` は省略可能にしない。該当しない設問でも空配列を入れる。
* `freeTextAnswer` の本文プロパティは `text` に統一する。
* `distributedCount` と `collectedCount` は未入力を許可するため `undefined` を許容する。
* `createdAt` と `updatedAt` はISO 8601文字列にする。
* `sortOrder` は画面表示順に使う。配列順も同じ順序に保つ。

### 5.3 新規アンケート初期値


## 6. IndexedDB設計

### 6.1 DB構成

| 項目 | 値 |
| --- | --- |
| DB名 | `survey_report_builder` |
| バージョン | `1` |
| Object Store | `surveys` |
| Key | `id` |

### 6.2 Repository API

`src/lib/surveyRepository.ts`

```ts
export async function listSurveys(): Promise<Survey[]>;
export async function getSurvey(id: string): Promise<Survey | null>;
export async function saveSurvey(survey: Survey): Promise<Survey>;
export async function deleteSurvey(id: string): Promise<void>;
export async function duplicateSurvey(id: string): Promise<Survey>;
```

### 6.3 保存時の処理

`saveSurvey` は以下を行う。

1. `updatedAt` を現在時刻に更新する。
2. 設問、選択肢、自由記述を `sortOrder` 昇順に並べる。
3. 空白だけの文字列を必要に応じて `trim` する。
4. IndexedDBに保存する。
5. 保存後の `Survey` を返す。

### 6.4 一覧取得時の並び順

`listSurveys` は `updatedAt` の降順で返す。

### 6.5 複製時の処理

`duplicateSurvey` は以下を行う。

* すべての `id` を新しく採番する。
* `title` の末尾に ` のコピー` を付ける。
* `createdAt` と `updatedAt` を現在時刻にする。

## 7. JSONエクスポート / インポート

### 7.1 MVP必須理由

IndexedDBはサーバーに送信しない点では安全だが、ブラウザのサイトデータ削除、端末変更、別ブラウザ利用ではデータを引き継げない。そのため、JSONエクスポート / インポートをMVP必須機能とする。

### 7.2 エクスポート形式

`src/lib/exportJson.ts`

```ts
export type SurveyExportFile = {
  appName: "survey_report_builder";
  schemaVersion: 1;
  exportedAt: string;
  survey: Survey;
};
```

### 7.3 ファイル名

```text
survey-report-YYYYMMDD-HHmmss.json
```

例：

```text
survey-report-20260701-213000.json
```

### 7.4 エクスポートAPI

```ts
export function buildSurveyExportFile(survey: Survey): SurveyExportFile;
export function downloadSurveyJson(survey: Survey): void;
```

### 7.5 インポートAPI

`src/lib/importJson.ts`

```ts
export async function readSurveyJsonFile(file: File): Promise<Survey>;
export function parseSurveyJson(jsonText: string): Survey;
```

### 7.6 インポート検証

インポート時は以下を検証する。

* JSONとして読めること
* `appName` が `survey_report_builder` であること
* `schemaVersion` が `1` であること
* `survey` が存在すること
* `survey.title` が文字列であること
* `survey.questions` が配列であること
* `questionType` が `single_choice`、`multiple_choice`、`free_text` のいずれかであること
* `options` が配列であること
* `freeTextAnswers` が配列であること
* `count` が0以上の整数であること

### 7.7 ID重複対応

インポートした `survey.id` が既存データと重複する場合は、新しいIDを採番して保存する。

設問、選択肢、自由記述のIDも、インポート時にすべて再採番してよい。ユーザーに見えるIDではないため、衝突回避を優先する。

## 8. 計算仕様

### 8.1 回収率

```ts
export function calculateCollectionRate(
  distributedCount?: number,
  collectedCount?: number
): number | null {
  if (distributedCount == null || distributedCount <= 0) return null;
  if (collectedCount == null || collectedCount < 0) return null;
  return (collectedCount / distributedCount) * 100;
}
```

### 8.2 単一選択の割合

単一選択では、選択肢ごとの回答数合計を分母にする。

```ts
export function calculateSingleChoiceRate(
  optionCount: number,
  totalOptionCount: number
): number | null {
  if (totalOptionCount <= 0) return null;
  return (optionCount / totalOptionCount) * 100;
}
```

### 8.3 複数選択の割合

複数選択では、回収数が入力されている場合だけ回収数を分母にする。

回収数が未入力または0の場合、割合は表示しない。選択肢回答数の合計を代替分母にはしない。

```ts
export function calculateMultipleChoiceRate(
  optionCount: number,
  collectedCount?: number
): number | null {
  if (collectedCount == null || collectedCount <= 0) return null;
  return (optionCount / collectedCount) * 100;
}
```

### 8.4 表示丸め

```ts
export function formatRate(rate: number | null): string {
  if (rate == null) return "-";
  return `${rate.toFixed(1)}%`;
}
```

### 8.5 グラフ幅

横棒グラフの幅は、割合が取得できる場合は `0%` から `100%` に丸め込む。

```ts
export function clampRateForBar(rate: number | null): number {
  if (rate == null) return 0;
  return Math.max(0, Math.min(100, rate));
}
```

複数選択では割合が100%を超えることは通常ないが、回答数が回収数を超える入力もあり得る。棒の表示幅は100%を上限にし、数値は実値を表示する。

## 9. 入力チェック

### 9.1 エラーと警告の扱い

* エラーは保存を止める。
* 警告は保存を止めない。
* 警告は編集画面とプレビュー画面の両方で表示する。

### 9.2 エラー条件

| 条件 | メッセージ |
| --- | --- |
| タイトルが空 | タイトルを入力してください。 |
| 設問文が空 | 設問文を入力してください。 |
| 選択式の選択肢が0件 | 選択肢を1件以上入力してください。 |
| 選択肢名が空 | 選択肢名を入力してください。 |
| 回答数が未入力 | 回答数を入力してください。 |
| 回答数が0未満 | 回答数は0以上の整数で入力してください。 |
| 配布数が0未満 | 配布数は0以上の整数で入力してください。 |
| 回収数が0未満 | 回収数は0以上の整数で入力してください。 |

### 9.3 警告条件

| 条件 | メッセージ |
| --- | --- |
| 回収数が配布数を超えている | 回収数が配布数を超えています。内容をご確認ください。 |
| 単一選択の回答数合計が回収数と一致しない | 単一選択の回答数合計が回収数と一致していません。 |
| 複数選択の回答数合計が回収数を超えている | 複数回答のため、回答数合計が回収数を超える場合があります。 |
| 選択式の回答数がすべて0 | 回答数が入力されていない可能性があります。 |
| 自由記述が非常に長い | 印刷時にページ数が多くなる可能性があります。 |
| 自由記述設問に自由記述もまとめもない | 自由記述の内容が未入力です。 |

## 12. 印刷仕様

### 12.1 出力方式

PDFファイルはサーバーで生成しない。PDF出力ボタンでは以下を実行する。

```ts
window.print();
```

ユーザーはブラウザの印刷ダイアログで、PDF保存または紙印刷を選択する。

### 12.2 印刷CSS

`src/styles/print.css`

```css
@media print {
  @page {
    size: A4 portrait;
    margin: 12mm;
  }

  body {
    color: #000;
    background: #fff;
  }

  .no-print {
    display: none !important;
  }

  .print-page {
    width: 100%;
  }

  .question-block {
    break-inside: avoid;
    page-break-inside: avoid;
  }

  .page-break {
    break-before: page;
    page-break-before: always;
  }

  a {
    color: inherit;
    text-decoration: none;
  }
}
```

### 12.3 白黒印刷ルール

* 色だけで情報を表現しない。
* 件数と割合を必ず数値で表示する。
* 横棒グラフは枠線、濃淡、数値で判別できるようにする。
* 表は罫線を使う。
* 背景色に依存しない。
* 文字サイズは印刷時に読める大きさにする。
* 設問ブロックは可能な限り途中で分断しない。

## 13. UI方針

### 13.1 基本方針

* ITに詳しくない利用者でも使える画面にする。
* 専門用語を避ける。
* ボタン文言は操作結果が分かるものにする。
* 入力欄には必ずラベルを付ける。
* 印刷前にプレビューできるようにする。
* 自動計算の結果と入力値を混同しない表示にする。

### 13.2 ボタン文言

| 機能 | 表示文言 |
| --- | --- |
| 新規作成 | 新しいアンケートを作成 |
| 保存 | 保存 |
| プレビュー | レポートを確認 |
| 印刷 | PDF出力・印刷 |
| エクスポート | データを保存 |
| インポート | データを読み込み |
| 設問追加 | 設問を追加 |
| 選択肢追加 | 選択肢を追加 |
| 自由記述追加 | 自由記述を追加 |

### 13.3 注意文

全体注意：

```text
入力データはサーバーには保存されず、このブラウザ内に保存されます。
別の端末やブラウザでは表示されません。
ブラウザのデータを削除すると、保存内容も削除される場合があります。
必要に応じてJSONファイルとして保存してください。
```

自由記述欄の注意：

```text
個人名、住所、電話番号などの個人情報は、必要がない限り入力しないでください。
```

印刷案内：

```text
PDFとして保存する場合は、印刷ダイアログの送信先で「PDFに保存」を選択してください。
```

## 16. 手動確認シナリオ

### 16.1 基本シナリオ

1. 新しいアンケートを作成する。
2. タイトル、発行元、実施期間、配布数、回収数を入力する。
3. 単一選択の設問を追加する。
4. 複数選択の設問を追加する。
5. 自由記述の設問を追加する。
6. 保存する。
7. 一覧に戻る。
8. 保存したアンケートを開く。
9. レポートを確認する。
10. PDF出力・印刷を実行する。

期待結果：

* 保存内容が消えない。
* 回収率と割合が正しい。
* PDF保存できる。

### 16.2 JSONバックアップ

1. アンケートを作成して保存する。
2. データを保存からJSONをダウンロードする。
3. アンケートを削除する。
4. データを読み込みからJSONをインポートする。

期待結果：

* アンケートが復元される。
* 既存IDと重複しても保存できる。

### 16.3 入力チェック

1. タイトルなしで保存する。
2. 選択式設問で選択肢なしにする。
3. 回収数を配布数より大きくする。
4. 単一選択の回答数合計を回収数とずらす。

期待結果：

* タイトルなし、選択肢なしはエラーになる。
* 回収数超過、回答数不一致は警告になる。

## 18. 実装時の優先判断

迷った場合は以下を優先する。

1. 入力したデータを失いにくいこと。
2. 印刷したレポートが読みやすいこと。
3. ITに詳しくない利用者が迷わないこと。
4. サーバーへ入力データを送信しないこと。
5. 実装を過度に複雑にしないこと。

初期リリースでは、見た目の華やかさよりも、入力、保存、復元、印刷の確実性を優先する。
