# アンケート集計レポート 設計書

## 1. 概要

### 1.1 サービス名

アンケート集計レポート

### 1.2 開発名

survey_report_builder

### 1.3 目的

紙や手作業で集計したアンケート結果を、Web画面から入力し、A4縦向き・白黒印刷に適したPDF向けレポートとして出力できるようにする。

本サービスは、オンラインアンケートの回答収集を目的とするものではなく、すでに回収・集計されたアンケート結果を、見やすい配布用資料として整えることを目的とする。

### 1.4 想定する成果物

* A4縦向きのアンケート集計レポート
* 白黒印刷でも見やすい集計表
* 必要に応じた簡易横棒グラフ
* 自由記述一覧
* 自由記述まとめ
* PDF保存または紙印刷用の画面

---

## 2. サービス方針

## 2.1 基本方針

本サービスでは、ユーザーが以下の情報を入力する。

* アンケートタイトル
* 発行元または実施主体
* 実施期間
* 配布数
* 回収数
* 設問
* 選択肢
* 選択肢ごとの回答数
* 自由記述
* 自由記述まとめ

入力された内容をもとに、回答割合や回収率を自動計算し、PDF向けのレポート画面を生成する。

### 2.2 対象を特定組織に限定しない

本サービスは、町内会、自治会、PTA、管理組合など特定の組織に限定しない。

「どの団体が実施したアンケートか」は、発行元やタイトルで表現できるようにする。

そのため、画面やデータ項目では「町内会専用」といった固定表現は避け、汎用的なアンケート集計レポート作成ツールとして設計する。

| PWA対応     | 初期リリースでは対応しない    |

## 3.2 技術方針

| 項目        | 内容                        |
| --------- | ------------------------- |
| フロントエンド   | React + Vite + TypeScript |
| データ保存     | IndexedDB                 |
| PDF出力     | ブラウザ印刷機能                  |
| グラフ       | CSSによる簡易横棒グラフ             |
| データバックアップ | JSONエクスポート / インポート        |

## 3.3 保存方針

入力データはサーバーに保存しない。
利用者のブラウザ内に保存する。

保存先には IndexedDB を使用する。

IndexedDB はブラウザ内にデータを保存する仕組みであり、通常はブラウザを閉じてもデータは保持される。
ただし、以下の場合にはデータが失われる可能性がある。

* ユーザーがブラウザのサイトデータを削除した場合
* シークレットモードやプライベートブラウズを使用した場合
* 別端末・別ブラウザでアクセスした場合
* ブラウザやOSのストレージ管理によりデータが削除された場合
* ブラウザの不具合や初期化が発生した場合

そのため、JSON形式でのエクスポート / インポート機能を提供し、利用者が任意にバックアップできるようにする。

## 6. 設問仕様

## 6.1 対応する回答形式

初期リリースでは、以下の3種類に対応する。

| 回答形式 | 内容             |
| ---- | -------------- |
| 単一選択 | 1つの選択肢を選ぶ形式    |
| 複数選択 | 複数の選択肢を選べる形式   |
| 自由記述 | テキストで意見を記入する形式 |

---

## 6.2 単一選択

### 入力内容

* 設問文
* 選択肢
* 選択肢ごとの回答数

### 入力例

```text
設問：今後、防災訓練に参加したいと思いますか？
回答形式：単一選択

参加したい：32件
内容によって参加したい：41件
参加したくない：11件
```

### 表示仕様

単一選択では、選択肢ごとの回答数合計を分母として割合を算出する。

```text
割合 = 選択肢の回答数 ÷ 選択肢回答数の合計 × 100
```

小数第1位まで表示する。

---

## 6.3 複数選択

### 入力内容

* 設問文
* 選択肢
* 選択肢ごとの回答数

### 入力例

```text
設問：参加したい行事を選んでください。
回答形式：複数選択

防災訓練：40件
夏祭り：55件
清掃活動：30件
子ども会：20件
```

### 表示仕様

複数選択では、回答数の合計が回収数を超える可能性がある。

回収数が入力されている場合、割合は回収数を分母として算出する。

```text
割合 = 選択肢の回答数 ÷ 回収数 × 100
```

回収数が未入力の場合は、割合を表示しない、または選択肢回答数の合計を分母とする。

### 注記

複数選択の設問には、以下の注記を表示する。

```text
※複数回答のため、回答数の合計は回収数と一致しない場合があります。
```

---

## 6.4 自由記述

### 入力内容

* 設問文
* 自由記述一覧
* 自由記述まとめ

### 入力例

```text
設問：防災訓練についてご意見があればお書きください。

自由記述：
- 高齢者でも参加しやすい時間帯にしてほしい
- 子ども向けの内容も入れてほしい
- 休日の午前中が参加しやすい

自由記述まとめ：
参加しやすい時間帯への配慮や、子ども向け企画を求める意見が見られた。
```

### 表示仕様

自由記述は、入力順に箇条書きで表示する。

自由記述まとめが入力されている場合は、自由記述一覧の前に「主なご意見」または「自由記述まとめ」として表示する。

AIによる自動要約は初期リリースでは行わない。
まとめ欄はユーザーが任意で入力する。

## 8. PDF・印刷仕様

## 8.1 出力方式

本サービスでは、サーバー側でPDFファイルを生成しない。

レポート画面をHTML/CSSで生成し、ブラウザの印刷機能を利用してPDF保存または紙印刷を行う。

PDF出力ボタン押下時には、以下の処理を行う。

```text
window.print()
```

## 8.2 用紙設定

| 項目    | 内容                |
| ----- | ----------------- |
| 用紙サイズ | A4                |
| 向き    | 縦                 |
| 色     | 白黒印刷前提            |
| ページ数  | 内容に応じて複数ページ可      |
| 余白    | 標準または印刷用CSSで指定    |
| 改ページ  | 設問単位で極力分割されないよう調整 |

## 8.3 印刷レイアウト方針

* A4縦向きで読みやすい余白を確保する
* 白黒印刷でも判別できるようにする
* 色だけで情報を表現しない
* 件数と割合は必ず数値で表示する
* 簡易横棒グラフは罫線・濃淡・数値で補足する
* 自由記述は長文になりすぎないよう、改行と余白を調整する
* 内容量に応じて複数ページを許容する

## 10. 自動計算仕様

## 10.1 回収率

配布数と回収数が入力されている場合、回収率を自動計算する。

```text
回収率 = 回収数 ÷ 配布数 × 100
```

表示は小数第1位までとする。

### 例

```text
配布数：120
回収数：84
回収率：70.0%
```

---

## 10.2 単一選択の割合

単一選択では、選択肢ごとの回答数合計を分母として割合を算出する。

```text
割合 = 選択肢の回答数 ÷ 選択肢回答数の合計 × 100
```

表示は小数第1位までとする。

---

## 10.3 複数選択の割合

複数選択では、回収数が入力されている場合は回収数を分母として割合を算出する。

```text
割合 = 選択肢の回答数 ÷ 回収数 × 100
```

回収数が未入力の場合は、割合を表示しない。

---

## 10.4 自由記述

自由記述は自動計算を行わない。
入力された内容を、入力順に一覧表示する。

自由記述まとめはユーザーが任意で入力する。

| ------- | ------------- |

## 11.3 警告表示

以下の場合、保存は許可するが警告を表示する。

| 条件                   | 警告内容                      |
| -------------------- | ------------------------- |
| 回収数が配布数を超えている        | 回収数が配布数を超えています            |
| 単一選択の回答数合計が回収数と一致しない | 回答数の合計が回収数と一致していません       |
| 複数選択の回答数合計が回収数を超えている | 複数回答のため、合計が回収数を超える場合があります |
| 回答数がすべて0の設問がある       | 回答数が入力されていない可能性があります      |
| 自由記述が非常に長い           | 印刷時にページ数が多くなる可能性があります     |

## 15. JSONエクスポート / インポート仕様

## 15.1 エクスポート

アンケート単位でJSONファイルを出力する。

### ファイル名例

```text
survey-report-20260701.json
```

### 出力内容

* Surveyオブジェクト一式
* 設問
* 選択肢
* 回答数
* 自由記述
* 自由記述まとめ

## 15.2 インポート

JSONファイルを読み込み、アンケートデータとして復元する。

### インポート時のチェック

* JSON形式であること
* 必須項目が存在すること
* `questions` が配列であること
* `questionType` が定義済みの値であること
* 回答数が数値であること

### 既存データとの重複対応

インポートしたアンケートIDが既存データと重複する場合は、新しいIDを採番して保存する。

## 18. 計算ロジック

## 18.1 回収率計算

```ts
function calculateCollectionRate(
  distributedCount?: number,
  collectedCount?: number
): number | null {
  if (!distributedCount || distributedCount <= 0) return null;
  if (collectedCount == null || collectedCount < 0) return null;

  return (collectedCount / distributedCount) * 100;
}
```

## 18.2 単一選択の割合計算

```ts
function calculateSingleChoiceRate(
  optionCount: number,
  totalOptionCount: number
): number | null {
  if (totalOptionCount <= 0) return null;
  return (optionCount / totalOptionCount) * 100;
}
```

## 18.3 複数選択の割合計算

```ts
function calculateMultipleChoiceRate(
  optionCount: number,
  collectedCount?: number
): number | null {
  if (!collectedCount || collectedCount <= 0) return null;
  return (optionCount / collectedCount) * 100;
}
```

## 18.4 表示丸め

割合は小数第1位まで表示する。

```ts
function formatRate(rate: number | null): string {
  if (rate == null) return "-";
  return `${rate.toFixed(1)}%`;
}
```

## 19. セキュリティ・個人情報方針

## 19.1 サーバー保存なし

本サービスでは、入力データをサーバーに保存しない。

アンケート内容、自由記述、発行元、備考などは、利用者のブラウザ内に保存される。

## 19.2 注意表示

画面上に以下の注意文を表示する。

```text
入力データはサーバーには保存されず、このブラウザ内に保存されます。
別の端末やブラウザでは表示されません。
ブラウザのデータを削除すると、保存内容も削除される場合があります。
必要に応じてJSONファイルとして保存してください。
```

自由記述欄付近には以下の注意文を表示する。

```text
個人名、住所、電話番号などの個人情報は、必要がない限り入力しないでください。
```

Cloudflare Pagesで静的ファイルを配信する。
初期リリースでは、ユーザー入力データを外部APIへ送信しない。

## 27. 備考

本サービスは、集計業務そのものを完全自動化するものではない。

初期リリースでは、紙アンケートの内容を人が確認し、設問・選択肢・回答数を入力することを前提とする。

価値の中心は、以下の3点である。

* 集計結果の資料化を簡単にすること
* 白黒印刷でも見やすいPDF向けレポートを作成できること
* サーバーにデータを保存せず、固定費なしで公開・運用できること
