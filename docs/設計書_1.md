# アンケート集計レポート 設計書

## 1. 概要

### 1.1 サービス名

アンケート集計レポート

### 1.2 開発名

survey_report_builder

### 1.3 目的

紙などで回収したアンケートの集計結果を入力し、A4縦向き・白黒印刷に適したPDF向けレポートを作成する。

本サービスは、オンラインアンケートの作成・回答収集を目的としたものではなく、すでに回収済みのアンケート結果を、見やすい集計資料として整えることを目的とする。

### 1.4 想定する利用シーン

* 紙で配布・回収したアンケートを集計したい
* Excelでの整形やPDF化が面倒
* 回覧・配布・掲示用にA4資料を作りたい
* 設問や選択肢が毎回異なるため、固定テンプレートでは対応しにくい
* サーバーにアンケート内容を保存せず、利用者の端末内で完結させたい

## 3. サービス方針

### 3.1 基本方針

本サービスは、アンケート回答者1人ずつの回答内容を入力するのではなく、設問ごとに集計済みの回答数を入力する方式とする。

例：

```text
設問：今後の活動に参加したいと思いますか？

参加したい：32件
内容によって参加したい：41件
参加したくない：11件
```

入力された回答数をもとに、割合計算、表作成、簡易グラフ表示、PDF向けレイアウト作成を自動化する。

### 3.2 サービスの位置づけ

本サービスは、Google Formsのようなオンラインアンケート作成サービスではない。

主な価値は以下である。

* 紙で集めたアンケートを資料化できる
* 設問・選択肢・回答数を入力するだけで集計レポートを作れる
* 白黒印刷しやすいA4レポートを作れる
* サーバーにデータを保存しない
* 固定費をかけずに公開・運用できる

## 6. 回答形式

### 6.1 対応する回答形式

初期リリースでは以下の3種類に対応する。

| 回答形式 | 内容             |
| ---- | -------------- |
| 単一選択 | 1つの選択肢を選ぶ形式    |
| 複数選択 | 複数の選択肢を選べる形式   |
| 自由記述 | テキストで意見を記入する形式 |

## 7. 単一選択

### 7.1 入力内容

* 設問文
* 選択肢
* 選択肢ごとの回答数

### 7.2 表示内容

* 選択肢名
* 回答数
* 割合
* 簡易横棒グラフ

### 7.3 計算仕様

単一選択の場合、選択肢ごとの回答数合計を分母として割合を算出する。

```text
割合 = 選択肢の回答数 ÷ 選択肢回答数の合計 × 100
```

小数第1位まで表示する。

### 7.4 例

```text
1. 今後、活動に参加したいと思いますか？

参加したい                  32件  38.1%
内容によって参加したい      41件  48.8%
参加したくない              11件  13.1%
```

---

## 8. 複数選択

### 8.1 入力内容

* 設問文
* 選択肢
* 選択肢ごとの回答数

### 8.2 表示内容

* 選択肢名
* 回答数
* 割合
* 簡易横棒グラフ
* 複数回答である旨の注記

### 8.3 計算仕様

複数選択の場合、回収数が入力されていれば、回収数を分母として割合を算出する。

```text
割合 = 選択肢の回答数 ÷ 回収数 × 100
```

回収数が未入力の場合は、割合を表示しない。

### 8.4 注記

複数選択の設問には、以下の注記を表示する。

```text
※複数回答のため、回答数の合計は回収数と一致しない場合があります。
```

### 8.5 例

```text
2. 参加したい行事を選んでください。※複数回答可

防災訓練      40件  47.6%
夏祭り        55件  65.5%
清掃活動      30件  35.7%
子ども会      20件  23.8%
```

## 9. 自由記述

### 9.1 入力内容

自由記述形式の設問では、以下を入力できるようにする。

* 設問文
* 自由記述一覧
* 自由記述まとめ

### 9.2 自由記述一覧

回収した自由記述を1件ずつ入力する。

入力された自由記述は、PDF上では箇条書きで表示する。

### 9.3 自由記述まとめ

自由記述の内容をもとに、利用者が任意でまとめ文を入力できるようにする。

AIによる自動要約は初期リリースでは行わない。

### 9.4 表示例

```text

主なご意見：
開催時間については、午前中を希望する意見が複数ありました。
また、子ども向け行事を希望する声もありました。

・高齢者でも参加しやすい時間帯にしてほしい
・子ども向け行事があるとよい
・回覧板の頻度を減らしてほしい
```

## 11. PDF・印刷仕様

### 11.1 出力方式

PDFファイルをサーバーで生成しない。

ブラウザの印刷機能を利用し、利用者が印刷ダイアログから以下を選択する。

* PDFとして保存
* プリンターで印刷

アプリ上のボタン名は「PDF出力」とする。
内部処理では `window.print()` を呼び出す。

### 11.2 用紙設定

| 項目    | 内容                  |
| ----- | ------------------- |
| 用紙サイズ | A4                  |
| 向き    | 縦                   |
| 印刷色   | 白黒印刷前提              |
| ページ数  | 内容に応じて複数ページ可        |
| 余白    | 標準余白                |
| 改ページ  | 設問単位で不自然に分断されないよう調整 |

### 11.3 レポート構成

1. タイトル
2. 発行元
3. アンケート概要
4. 設問ごとの集計結果
5. 自由記述
6. 注記

### 11.4 白黒印刷対応

* 色だけで情報を表現しない
* 件数と割合を必ず表示する
* 横棒グラフは枠線・濃淡・数値で判別できるようにする
* 表は罫線を使用する
* 背景色に依存しない
* 文字サイズは印刷時に読みやすいサイズにする

### 11.5 印刷CSS

印刷用CSSを用意する。

```css
@media print {
  @page {
    size: A4 portrait;
    margin: 12mm;
  }

  .no-print {
    display: none;
  }

  .page-break {
    break-before: page;
  }

  .question-block {
    break-inside: avoid;
  }
}
```

## 12. グラフ仕様

### 12.1 表示方式

初期リリースでは、Chart.jsなどのグラフライブラリは使用しない。

HTML/CSSによる簡易横棒グラフを表示する。

### 12.2 理由

* 白黒印刷に対応しやすい
* 依存ライブラリを減らせる
* PDF出力時に崩れにくい
* 実装が軽い
* 件数と割合が主役なので、高度なグラフ表現は不要

### 12.3 表示例

```text
内容によって参加したい  41件  48.8%  [##########----]
参加したくない          11件  13.1%  [###-----------]
```

## 13. データ保存仕様

### 13.1 保存方式

IndexedDBを使用する。

IndexedDBはブラウザ内のローカル保存領域であり、サーバーにはデータを送信しない。

### 13.2 IndexedDB採用理由

* DBサーバーが不要
* 固定費がかからない
* localStorageより大きなデータを扱いやすい
* JavaScriptオブジェクトを保存しやすい
* 複数アンケートの管理に向いている
* 将来的に自由記述が増えても対応しやすい

### 13.3 永続性について

IndexedDBのデータは、通常のブラウザ利用では永続保存される。

ただし、以下の場合はデータが削除される可能性がある。

* 利用者がブラウザのサイトデータを削除した場合
* シークレットモード、プライベートブラウジングを利用した場合
* ブラウザやOSのストレージ管理により削除された場合
* 端末の空き容量が不足した場合
* 別ブラウザ・別端末で開いた場合

そのため、画面上に以下の注意文を表示する。

```text
入力データはこのブラウザ内に保存されます。
サーバーには保存されません。
ブラウザのデータ削除や端末変更時には、保存内容を引き継げない場合があります。
```

### 13.4 JSONバックアップ

初期リリースでは必須機能としない。

ただし、IndexedDBのデータが利用者環境に依存するため、将来的な拡張候補として以下を用意する。

* JSONエクスポート
* JSONインポート

## 15. 自動計算仕様

### 15.1 回収率

配布数と回収数が入力されている場合、以下で算出する。

```text
回収率 = 回収数 ÷ 配布数 × 100
```

表示は小数第1位までとする。

### 15.2 単一選択の割合

```text
割合 = 選択肢の回答数 ÷ 選択肢回答数の合計 × 100
```

表示は小数第1位までとする。

### 15.3 複数選択の割合

回収数が入力されている場合：

```text
割合 = 選択肢の回答数 ÷ 回収数 × 100
```

回収数が未入力の場合：

```text
割合は表示しない
```

### 15.4 0除算対応

分母が0または未入力の場合、割合は表示しない。

表示例：

```text
-
```

## 16. 入力チェック仕様

### 16.1 アンケート基本情報

| 項目   | チェック内容 |
| ---- | ------ |
| タイトル | 必須     |
| 発行元  | 任意     |
| 実施期間 | 任意     |
| 配布数  | 0以上の数値 |
| 回収数  | 0以上の数値 |
| 備考   | 任意     |

### 16.2 設問

| 項目      | チェック内容        |
| ------- | ------------- |
| 設問文     | 必須            |
| 回答形式    | 必須            |
| 選択肢     | 選択式の場合、1件以上必須 |
| 回答数     | 0以上の数値        |
| 自由記述    | 未入力可          |
| 自由記述まとめ | 未入力可          |

### 16.3 警告

以下は保存を禁止せず、警告表示に留める。

* 回収数が配布数を超えている
* 単一選択の回答数合計が回収数と一致しない
* 複数選択の回答数合計が回収数を超えている
* 回答数がすべて0の設問がある
* 自由記述設問に自由記述が1件もない

### 16.4 警告メッセージ例

```text
単一選択の回答数合計が回収数と一致していません。
内容をご確認ください。
```

```text
複数回答のため、回答数合計が回収数を超える場合があります。
```

### 17.2 注意文

### 17.3 自由記述への注意

自由記述には個人情報や機微情報が含まれる可能性があるため、自由記述入力欄の近くにも注意文を表示する。

```text
自由記述に個人名・住所・電話番号などが含まれていないか確認してください。
```

### 18.3 Cloudflare Pages設定

React / Vite構成を想定する。

Cloudflare PagesのReact向け公式ガイドでは、GitHub連携時の設定として Production branch は `main`、Build command は `npm run build`、Build directory は `dist` が示されているため、本サービスでも同設定を基本とする。

```text
Production branch: main
Build command: npm run build
Build directory: dist
```

## 24. 非機能要件

### 24.1 固定費

固定費をかけない。

Cloudflare Pagesの無料利用範囲で公開する。

### 24.2 パフォーマンス

* 初回表示が軽いこと
* 画面遷移が軽いこと
* 設問数が増えても編集操作が重くなりすぎないこと
* 印刷プレビューが崩れないこと

### 24.3 ブラウザ対応

初期リリースでは、最新版の主要ブラウザを対象とする。

* Google Chrome
* Microsoft Edge
* Safari
* Firefox

### 24.4 アクセシビリティ

* 入力欄にラベルを付ける
* ボタン名をわかりやすくする
* 色だけで情報を表現しない
* キーボード操作を大きく妨げない

## 28. 備考

本サービスは、アンケート回答の収集ではなく、アンケート集計結果の資料化に特化する。

初期リリースでは、機能を増やしすぎず、以下の価値に集中する。

```text
設問を入力する
選択肢と回答数を入力する
自由記述を入力する
見やすいA4レポートにする
PDFとして保存・印刷する
```

サーバー保存を行わないため、利用者のプライバシー面では扱いやすい一方、ブラウザデータ削除時の消失リスクがある。
この点は画面上で明示し、将来的にはJSONエクスポート・インポート機能で補完する。
