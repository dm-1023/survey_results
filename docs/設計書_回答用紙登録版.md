# アンケート集計レポート 回答用紙登録版 設計メモ

## 1. 方針変更

当初の「集計済み件数を入力する方式」から、「回答用紙を1件ずつ登録し、その回答データから集計・出力する方式」へ変更する。

この方式により、以下を実現する。

* 入力後に何度でも集計し直せる
* 問4のようなマトリクス形式を自然に扱える
* 回答データと連絡先データを分離できる
* 集計レポートには個人情報を出さない設計にできる
* 将来的にCSV出力、Excel利用、クロス集計へ拡張しやすい

## 2. データ分離

### 2.1 回答データ

問1〜問8、問10、問11を回答データとして保存する。

回答データは集計対象とする。

### 2.2 連絡先データ

問9の氏名、住所、電話番号は連絡先データとして保存する。

連絡先データは回答IDと紐づけるが、集計レポートでは読み込まない。

集計レポートに表示するのは、必要な場合でも「協力可能者数」のみとする。

## 3. IndexedDB構成

| Store | 内容 |
| --- | --- |
| responses | 回答用紙1件ごとの回答 |
| contacts | 問9の連絡先 |

## 4. 出力方針

| 出力 | 内容 | 用途 |
| --- | --- | --- |
| 集計レポート | 集計値のみ。連絡先なし | 配布・共有用 |
| 匿名回答CSV | 連絡先を除いた回答データ | Excel集計・検証用 |
| 集計CSV | 各設問の集計結果 | Excel資料化用 |
| 連絡先CSV | 問9のみ | 運営内部用 |
| 完全バックアップJSON | 回答と連絡先を含む | 復元・保管用 |

匿名回答CSVにも自由記述は含まれるため、共有前に個人情報が含まれていないか確認する。

## 5. レポートに出さない情報

以下は配布用PDFや集計レポートには出さない。

* 氏名
* 住所
* 電話番号
* 回答用紙ID
* 連絡先と回答内容を結びつける情報

自由記述は個人情報が含まれる可能性があるため、初期表示では件数のみを出す。
原文を利用する場合は、運営側が内容を確認し、個人が特定できる表現を伏せた上で資料化する。

## 6. 今回実装する画面

* アンケート一覧トップ
* アンケート設定作成・編集
* 回答一覧
* 回答入力・編集
* 集計レポート
* 連絡先一覧

画面導線は以下とする。

```text
アンケート一覧トップ
  -> アンケートを選択
  -> 回答一覧
  -> 回答入力 / 集計レポート / 連絡先管理
```

アンケート一覧トップでは、アンケートの作成、設定編集、削除を行う。
回答、連絡先、集計、CSV出力は選択中アンケート単位で行う。

## 7. 今回実装する出力

* 匿名回答CSV
* 集計CSV
* 連絡先CSV
* 完全バックアップJSON
* 完全バックアップJSONの読み込み
